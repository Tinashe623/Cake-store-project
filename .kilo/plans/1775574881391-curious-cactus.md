# Multi-Language Support (English & Shona) Implementation Plan

## Overview
Add internationalization (i18n) support to the Tarie Cakes website using i18next and react-i18next, enabling English and Shona language switching via a toggle button in the UI.

## Dependencies
Install required packages:
```bash
npm install i18next react-i18next i18next-browser-languagedetector
```

## Implementation Steps

### 1. Setup i18n Configuration
Create `src/i18n.ts`:
- Initialize i18next with language detector
- Configure English (en) as default language
- Configure Shona (sn) as secondary language
- Load translations from JSON files

### 2. Create Translation Files
Create `public/locales/` directory with:
- `en.json` - English translations
- `sn.json` - Shona translations

Initial translation keys needed:
- Navigation: Home, Menu, Gallery, About, Contact
- Hero: badge, heading, subtitle, cta buttons
- Header: Order Now button
- Footer: sections (Explore, Get in Touch, Newsletter), links, copyright
- Common: email placeholder, submit buttons, etc.

### 3. Create Language Switcher Component
Create `src/components/LanguageSwitcher.tsx`:
- Simple toggle button showing "EN | SN" or current language
- Position in Header next to nav links (desktop) or in mobile drawer
- Uses i18n's `changeLanguage()` function

### 4. Update Header Component
Modify `src/components/Header.tsx`:
- Import and add LanguageSwitcher component
- Add to desktop nav (after navLinks) and mobile drawer

### 5. Update Components with Translations
Wrap static text with `useTranslation()` hook:
- **Header.tsx**: navLinks, Order Now button
- **Footer.tsx**: all static text
- **Hero.tsx**: badge, heading parts, subtitle, buttons
- **HowItWorks.tsx**: section text
- **Contact.tsx**: section headers, form labels
- **About.tsx**: static content
- **Gallery.tsx**: headers
- **Testimonials.tsx**: headers

### 6. Update App Entry Point
Modify `src/main.tsx` (or `index.tsx`):
- Import i18n configuration
- Initialize i18n before rendering App

## File Structure After Implementation
```
public/
  locales/
    en.json
    sn.json
src/
  i18n.ts
  components/
    LanguageSwitcher.tsx
```

## Notes
- Translation keys use dot notation for nesting (e.g., `nav.home`, `hero.title`)
- Dynamic content like cake names/descriptions from data files will be handled separately
- Browser language detection will auto-set initial language
- Language preference will persist via localStorage
